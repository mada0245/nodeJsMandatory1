1 - SSR vs CSR

SSR

+

Faster in slow devices: because the initial HTML is loaded from the server

Better accessibility

Can provide better SEO results because search engines can crawl and index the pages more easily

-

Increased server load

Pages that requiere a round trip to the server may have a slower load

Can be more complex to implement and maintain because it requires additional server-side configuration and setup

CSR

+

Faster page loads because the initial HTML is loaded fast and the JavaScript code takes over to render the page dynamically

It allows for dynamic updates without the need for page refreshes

Easier and more flexible maintenance and development

-

SEO challenges, search engines may not be able to crawl and index the pages properly

Performance issues on slow devices

CSR can be less accessible because the initial HTML is not available to users who have limited access to JavaScript



2 - What couses XSS and how sanitaize using npm package

XSS occurs when an someone is able to insert malicious code into a web page that is then executed by a user's browser

This can happen in many ways, like inserting code into form fields, URL parameters, or cookies

Using npm package

("npm install xss") -> install xss package

In the js file:

("const xss = require('xss');")

Then use the xss fuction to sanitaize the inputs

(" const input = "alert('XSS PROBLEM')"; const sanitaizedInput = xss(input); console.log(sanitaizedInput); ")
